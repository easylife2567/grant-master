# 第 01 轮综合理解报告

## 本轮新增核心理解

### 1. 领域全景的六个方向

经过 25 篇论文的精读（5 批次并行），本领域的全景图已清晰地呈现为六个方向：RDMA 微架构隔离（方向一）、软件抢占式隔离（方向二）、分布式训练通信调度（方向三）、拓扑感知与多任务隔离（方向四）、ML-感知 RDMA 传输新范式（方向五）、拥塞控制与负载均衡公平性（方向六）。

**证据→分析→影响**：25 篇论文的跨批次发现一致指向一个结构性的认知：该领域的"三层隔离体系"中，Layer 1（通信域安全隔离——Slingshot）和 Layer 2（性能资源隔离——Harmonic/PeRF/Palladium）已有成熟方案，但 Layer 3（**工作负载感知的动态资源调度**）在所有论文中都是明确的空白。这为本课题的定位提供了清晰的"图层"参照——课题瞄准的是 Layer 3，而不需要重新发明 Layer 1 和 Layer 2。影响：这意味着课题的技术方案可以"站在巨人肩上"，使用 PeRF 的抢占原语或 Harmonic 的 PIPS 监控作为基元，而创新点集中在"如何根据训练通信模式动态配置这些基元"。

### 2. Gap 的"交叉空白"确认

首轮 synthesis 最重要的发现是 **Gap 1 和 Gap 2 的"交叉空白"属性**——这不是一个"还没人做过"的空白，而是一个"两个成熟的子领域各自发展到了一定阶段但从未交汇"的空白。

**证据→分析→影响**：RDMA 隔离社区（6 篇核心）的论文作者主要来自网络系统背景，实验负载使用 Redis/memcached/perftest 等通用 benchmark——他们的设计假设是"应用流量是稳态随机流"。训练通信调度社区（7 篇核心）的作者主要来自 ML 系统背景，实验使用 PyTorch/TensorFlow 的训练脚本——他们的设计假设是"网络是足够快且无争用的管道"。两个社区各自的研究范式使得"训练通信的多租户网络隔离"这个自然的问题从未被系统性地提出。

影响：这个"交叉空白"属性直接提升了课题的创新性论据质量——不是说"这个方向没人做"（容易被质疑为"因为不值得做"），而是说"两个各自成熟的社区的自然交汇点恰好落在本课题的定位上，而这个交汇点之所以未被覆盖是因为两个社区的研究范式存在天然壁垒"。

### 3. 技术路线的三种选择

基于 25 篇论文的技术启发，识别出三条候选技术路线：

**证据→分析→影响**：
- **路线 A（纯软件训练感知调度）**：证据来自 PeRF（提供抢占原语，证明纯软件隔离可行）+ DLCP（提供 gradient-aware 范式，证明训练语义 → 网络层决策有效）+ BandPilot（提供学习带宽模型，证明智能预测可行）。可行性最高，但需要证明"训练通信模式感知 + 已有抢占原语 > 已有抢占原语"。
- **路线 B（拓扑感知协同放置 + 网络调度）**：证据来自 Arnold（MIP 优化框架）+ vClos（独占子拓扑）+ Ethereal（确定性路径分配）。性能可能最优但复杂度最高，需集成集群调度器。
- **路线 C（ML-容忍度驱动的差异化 QoS）**：证据来自 OptiNIC/Celeris（ML 容忍度定量）+ DLCP（bounded loss tolerance）+ P3（参数级差异化）。创新性最强但最激进——可能需要修改 NCCL 以暴露通信优先级接口。

影响：这三条路线不是互斥的——它们可以按优先级组合。对于青年项目，建议以路线 A 为主路径（纯软件、可验证、风险可控），路线 C 的子集（ML-容忍度驱动的 QoS 分类）作为增强维度。helm 需要从这三条中做最终收敛。
