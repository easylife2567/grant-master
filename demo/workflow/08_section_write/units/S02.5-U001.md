## 特色与创新

本项目的特色与创新主要体现在以下三个方面。

**创新点一：首次提出训练通信语义驱动的RDMA多租户调度范式。** 现有RDMA多租户性能隔离方案——无论是以Husky和Harmonic为代表的微架构感知路线、以PeRF为代表的软件抢占路线、还是以Palladium为代表的DPU卸载路线——均将上层应用流量视为无差别的稳态随机流，不感知也不利用分布式训练通信的确定性特征。本项目首次提出将训练通信语义（迭代周期、burst与空闲阶段的交替规律、通信操作类型差异、ML对部分数据丢失的容忍度）作为RDMA网络资源调度的核心决策变量，突破了现有方案的"信息不可知"局限。这一范式创新的本质是将训练层面对通信行为的理解与网络层面对资源分配的执行进行有机桥接，属于两个成熟子领域自然交汇产生的新方向。

**创新点二：设计了通信语义感知的差异化RDMA资源分配算法。** 不同于现有方案基于消息大小的粗粒度分类（如PeRF将消息按大小分三档）或基于历史流量的统计推断（如BandPilot的学习带宽模型），本项目提出基于通信操作语义的优先级映射和两级权重分配算法。该算法在QP粒度上区分梯度同步（高优先级保障带宽）、专家并行通信（中优先级弹性带宽）和数据加载（低优先级尽力而为带宽），并利用ML训练的丢包容忍度实施差异化降级——在有文献数据支撑的范围内（训练可容忍10%以内丢包而不影响收敛），允许低优先级任务在拥塞时接受"可控质量下降"以释放带宽给关键任务。表1展示了本项目与现有代表性方案在"是否感知训练通信模式"和"是否支持多租户动态调度"两个核心维度上的对比。

| 方案 | 隔离机制 | 是否感知训练通信模式 | 是否支持多租户动态调度 | 硬件需求 |
|------|---------|-------------------|---------------------|---------|
| Husky (NSDI'23) | 微架构诊断 | 否 | 否 | 无 |
| Harmonic (NSDI'24) | FPGA辅助限速 | 否 | 否 | FPGA |
| PeRF (ATC'24) | 软件抢占 | 否 | 否 | 无 |
| Palladium (SIGCOMM'25) | DPU反向代理 | 否 | 部分（加权轮询） | DPU |
| 本项目 | 训练语义驱动调度 | 是 | 是 | 无（纯软件） |

**创新点三：纯软件实现，不改NIC硬件，可增量部署于现有RoCEv2集群。** 与Harmonic需要FPGA、Palladium需要DPU、OptiNIC需要重新设计NIC传输层的硬件依赖路线不同，本项目的全部调度逻辑运行于用户空间的守护进程中，通过标准RDMA verbs和sysfs接口与现有RDMA协议栈交互，不需要修改NIC硬件或驱动。这一设计选择在保持技术先进性的同时，大幅降低了方案的部署门槛和实验验证难度——任何运行标准OFED驱动和NCCL的RoCEv2集群均可部署。此外，方案的回退机制确保了即使调度器出现故障，训练任务也不会中断，满足生产环境的可靠性要求。
